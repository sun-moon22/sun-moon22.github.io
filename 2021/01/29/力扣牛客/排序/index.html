一、简单排序

借鉴jdk的形式，在设计算法结构之前，先设计一套API，在对其方法一一实现。

如：

1.1 Comparable接口介绍

java开发中，常比较两个java对象。Comparable接口用来定义排序规则，我们对其进行实现即可。

可以对所有实现Comparable接口的对象进行排序。

例如：定义一个类User，实现Comparable接口，定义比较规则；定义测试类，并定义测试方法：Comparable getMax（Comparable c1,Comparable c2）；

1.2冒泡排序（Bubble sort）

1.2.1 原理

  1. 比较相邻元素。如果前一个大就交换两元素位置。

  2. 最终从小到大排序。

  3. 进行 数组长度-1 次大循环

  4. 优化： 某次循环中若没有发生交换，则已经有序，直接结束剩下的次数。

private static boolean greater(Comparable c1，Comparable c2); 判断c1是否大于c2

1.2.3 时间复杂度

O（n^2） 因此，待排序数较少时候可以，当数量较大时候，效率较低。

1.3选择排序（Selection sort）

1.3.1 原理

  1. （为每个位置找最小值）假设第一个索引为： 最小值索引 ，进行比较， **找出最小索引 ** 。
  2. **交换 ** 第一个索引处的值和最小值所在索引处的值。

1.4插入排序（Insertion sort）

1.4.1 原理

  1. 元素分组：已排序，未排序。
  2. 未排序第一个元素向已排序组中插入。
  3. 倒叙遍历已排序元素，依次和待插入元素比较，若待插元素小，交换。直到找到一个元素小于等于待插元素，停止交换，即插入成功。

二、高级排序

简单排序的时间复杂度均为O（n^2），随这输入规模的增大，时间将急剧上升，不能处理大规模问题。

2.1希尔排序（Shell Sort）

2.1.1 原理（插入排序的改良）

  1. 选定增长量h,按照h进行数组分组（h组数据）。

  2. 每一组数组完成 直接插入排序。

  3. 减小增长量，重复第2步，直到增长量为1。

2.2归并排序

2.2.1递归

定义：方法内部调用方法本身。

作用：将大型复杂真题转换成与原问题相似的小规模问题求解，用少量程序描述多次重复计算。

注意：不能无限制调用自己，应该设置边界条件，防止发生栈溢出（每调用一次，栈中开辟一块空间）。

例如：用代码实现n的阶乘。

2.2.2归并排序（Merge Sort）

分治法的典型应用，将已有序的子序列合并。

原理

  1. 将数组等额拆分，直到每个组仅有一个元素为止。

  2. 相邻自组合并，直到最终仅有一个组为止（边合并边排序，归并后的元素要有序）。

    3.归并原理：将归并后的数据放在辅助数组，通过3个数组来完成。比较上面两指针的元素，将较小的放在辅助数组中，并且被放的元素的指针后移。

API设计

类名 Merge
构造方法 Merge()
成员方法 public static void sort(Comparable[] a); 对数组内的元素进行排序
private static void exch(Comparable[] a, int i, int j); 没有用到。交换数组a中索引 i 和 j 的值
private static boolean greater(Comparable c1，Comparable c2); 判断c1是否大于c2
private static void sort（Comparable[] a, int lo, int hi）；对数组中索引lo到hi的元素进行排序
private static void merge(Comparable[] a, int lo, int mid, int hi);索引lo到mid为有一个自组；mid+1到hi为另外一个自组。将a的两个子组合并为有序的大组。
成员变量 private static Comparable[] assist;完成归并操作的辅助数组

时间复杂度

O（nlogn） 效率较高。

缺点：需要申请额外的数组空间，空间复杂度提升，典型的空间换时间的操作。

希尔排序和归并排序比较

通过事后分析。二则的效率差不多，要根据算法的稳定性进行抉择。

2.3快速排序

2.3.1 原理（插入排序的改良）

  1. 设定分界值，将数组分为左右两部分。
  2. 大于等于分界值的放在右边，小于的放在左边。
  3. 对于左右数组重复步骤2.
  4. 将递归排序后的数组进行合并。

![](C:\Users\Mr Li\AppData\Roaming\Typora\typora-user-images\image-20210718151311138.png)

切分原理：

  1. 确定基准值（数组第一个），用两个指针分别指向头部和尾部的下一位。

  2. 先从尾部开始搜索一个比基准值小的元素，搜索到即停止，并记录指针的位置；

  3. 再从头部像尾部搜索一个比基准值大的元素，搜索到即停止，并记录指针的位置；

  4. 交换当前左右边指针上的元素。

  5. 重复上步骤，直到left>=right。

  6. 交换基准值和当前两指针指向的元素。

    注意：当切分不充分时候，分界值左边的元素：小于等于分界值；分界值右边的元素：大于等于分界值。

2.3.3 时间复杂度

快速排序一次切分从两头开始交替搜素，直到left和right重合，因此，一次切分算法的时间复杂度为O（n），整个时间复杂度取决于切分次数。

最优情况：每一次切分选择的基准数子刚好将当前序列等分。

上图共切分logn此，所以最优情况下：O(nlogn)

最坏情况：每一次选择的基准数字是当前序列的最大数或最小数，每次切分都只有1个子组，那么一共切割n次，因此最坏情况：O(n^2)![image-20210718201742121](C:\Users\Mr Li\AppData\Roaming\Typora\typora-user-images\image-20210718201742121.png)

平均情况：O（nlogn）

2.3.4快速排序和归并排序的区别

相同：都是分治算法

区别：

  1. 归并排序将数组分成两个数组分别排序，将有序的子数组归并；快速排序是当两个数组有序时，整个数组就有序，无需归并。
  2. 切分：归并排序是根据lo和hi等分两半；而快速排序切分的位置取决于数组内容；

2.4排序的稳定性

稳定性

相等元素的相对位置不发生变化，则稳定。

![image-20210718202005704](C:\Users\Mr Li\AppData\Roaming\Typora\typora-user-images\image-20210718202005704.png)

**意义 **

仅针对多次排序的情况来说。保证第一次排序的原有意义，并减少系统开销。例如：第一次排序按照商品价格排序，第二次按照销量排序，对于销量相同的产品，其相对位置不发生变动，则之前按照价格排序的意义就可以保持不变。

几种算法的稳定性

稳定：冒泡排序、插入排序、归并排序

不稳定：选择排序、希尔排序、快速排序
